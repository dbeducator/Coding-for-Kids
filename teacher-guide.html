🧑‍🏫 Before You Begin — Key Tips

Start with the app, togetherDo the first lesson as a whole class on the projector/screen before learners work individually or in pairs.
Unplugged firstEven if you have robot kits, do the unplugged version first — it builds the mental model before adding hardware complexity.
Praise process, not answersSay "I like how you planned that step by step" not just "well done!" — builds computational thinking habits.
XP motivatesRemind learners about their XP and badges — let high-XP learners help others. Peer teaching is powerful.
Login is simpleLearners pick their class and name — no passwords. For Grade 00–2, teacher or helper can handle login.
All progress is savedEven if a lesson is interrupted, progress already logged to the sheet is safe. Learners continue where they left off.

Coding for Kids

Computational thinking, sequences & loops — no prior coding experience needed

Week 1 · Lesson 1

What is a Computer?

Duration: 30 min
Grade focus: Grade R–6
Materials: Devices with app open, optional whiteboard

Learning Objectives

  • Define a computer as a machine that follows instructions
  • Give 3 real-world examples of computers in everyday life
  • Explain why computers can't think for themselves

📝 Unplugged Activity

  • Ask: "What machines do you see that follow instructions?" (fans, traffic lights, microwaves)
  • Play "Robot Teacher" — teacher gives exact instructions, learners must follow literally (step forward, turn 90°, etc.)
  • Debrief: what happens when instructions are unclear?

💻 On the App

  • Open app → Coding for Kids → Lesson 1
  • Read lesson text together as a class
  • Learners answer the quiz individually
  • Discuss wrong answers as a group — never shame
Check for understanding: Ask 3 learners to name a computer they've seen that isn't a phone or laptop.
Week 1 · Lesson 2

Sequencing: Order Matters

Duration: 35 min
Grade focus: Grade R–6
Materials: Devices with app, index cards (optional)

Learning Objectives

  • Define a sequence as a set of steps in a specific order
  • Identify what goes wrong when steps are out of order
  • Arrange a real-world sequence correctly

📝 Unplugged Activity

  • Give groups of 4 index cards with scrambled steps ("making a sandwich" or "washing hands")
  • Groups race to arrange cards in the correct order
  • Act out the wrong order first — learners love the silliness of putting on shoes before socks!

💻 On the App

  • Open app → Coding for Kids → Lesson 2
  • Tap the steps in the correct sequence to complete the drag activity
  • Discuss: what would happen if you rinsed your mouth BEFORE brushing?
Check for understanding: Exit slip — each learner writes (or draws) 3 steps for getting dressed in the correct order.
Week 2 · Lesson 1

What is a Loop?

Duration: 30 min
Grade focus: Grade 1–6 (Grade R with support)
Materials: Devices with app

Learning Objectives

  • Define a loop as repeating instructions a set number of times
  • Identify loop patterns in everyday life
  • Explain why loops make code shorter and less error-prone

📝 Unplugged Activity

  • Whole class: clap 4 times. Now say "clap, repeat 4 times" — same result, one instruction!
  • Give a body movement sequence ("jump, spin, jump, spin, jump, spin") and ask: how many instructions? Can we use a loop?
  • Invent your own class loop sequence together

💻 On the App

  • Open app → Coding for Kids → Lesson 3
  • Read the loop explanation together
  • Learners answer the quiz — discuss correct answer as a class
Check for understanding: Ask: "If I need to write 'jump' 10 times, how can a loop help?" Learners explain to a partner.
Week 2 · Lesson 2

Unplugged Robot Game

Duration: 40 min
Grade focus: Grade R–6
Materials: Open floor space, tape for a grid (optional)

Learning Objectives

  • Combine sequence and loop concepts into a movement programme
  • Act as both programmer and robot to understand both perspectives
  • Debug a sequence when something goes wrong

📝 Unplugged Activity

  • Pairs: one is the Robot, one is the Programmer
  • Programmer writes a sequence on paper using only: "Forward 1, Turn Right, Turn Left, Stop"
  • Robot follows EXACTLY — even wrong instructions!
  • Switch roles after 3 minutes
  • Debrief: what was hard about writing clear instructions?

💻 On the App

  • Open app → Coding for Kids → Lesson 4
  • Plan the robot's sequence on screen
  • Drag moves into the dropzone in the correct order
  • Mark complete when finished — XP and possible badge awarded!
Check for understanding: Class discussion — "What is the hardest part of giving instructions to a robot that can't think for itself?"

Robotics Explorers

Sense-think-act, path planning and physical/digital integration — works with or without robot kits

Week 1 · Lesson 1

What is a Robot?

Duration: 30 min
Grade focus: Grade R–6
Materials (Unplugged): None · (Robot): 1 kit per 4–5 learners

Learning Objectives

  • Define a robot using the Sense → Think → Act framework
  • Identify robots in everyday settings (school, home, workplace)
  • Distinguish between a robot and a non-programmable machine

📝 Unplugged Version

  • Ask: "What is the difference between a fan and a robot vacuum?" (Fan just blows, robot navigates around furniture)
  • Sense-Think-Act role play: learner covers eyes (sensing), teacher whispers a rule (thinking), learner acts on it
  • Brainstorm: where do you think robots will work in Eswatini in 10 years?

🤖 With Kit Version

  • Pass robot kit around — don't turn on yet
  • Groups identify: what sensors does this robot have? (light sensors, bumpers, cameras?)
  • What does it "think" with? (its internal programme)
  • What does it "act" with? (wheels, lights, sounds)
Check for understanding: Learners draw a robot and label its Sense, Think and Act parts.
Week 1 · Lesson 2

Plan a Path

Duration: 40 min
Grade focus: Grade 1–6
Materials (Unplugged): Chalk/tape 4x4 grid on floor · (Robot): Floor robot + grid

Learning Objectives

  • Understand that robots need pre-planned movement instructions
  • Translate a visual grid path into a sequence of directional commands
  • Test and debug a planned sequence

📝 Unplugged Version

  • Draw a 4x4 grid on the floor with chalk or tape
  • Mark START and GOAL squares
  • One learner is the Robot — must follow spoken commands exactly
  • Class votes on each command before the Robot steps
  • Also complete the in-app sequence activity

🤖 With Kit Version

  • Set up floor robot on START square
  • Groups plan the path on the app FIRST — then press the exact same buttons on the robot
  • If the robot doesn't reach the goal, find the bug and try again
  • "Debugging" = fixing your instructions, not your robot!
Check for understanding: Ask groups to explain in words why they chose their sequence of moves.
Week 2 · Lesson 1

Loops Make It Easier

Duration: 30 min
Grade focus: Grade 1–6
Materials: Open space for movement · (Robot): Kit with repeat function

Learning Objectives

  • Apply loop concepts from Coding module to physical robot movements
  • Write a square-tracing programme using a repeating pattern
  • Appreciate that loops reduce both typing and errors

📝 Unplugged Version

  • Stand up: turn 90° right, 4 times — you're back where you started! That's a loop.
  • Now: forward 1, turn right. Repeat 4 times. You've traced a square.
  • Write this out on paper: "Forward 1, Turn Right" × 4 vs writing it out 8 times separately

🤖 With Kit Version

  • Challenge: programme the robot to trace a perfect square on the floor grid
  • If the kit has a repeat/loop button, use it. If not, press the same sequence 4 times
  • Discuss: why do robot programmers prefer loops?
Check for understanding: Learners write the instructions for a triangle (3 sides) — first without a loop, then with one.
Week 2 · Lesson 2

Robot Challenge: Obstacle Course

Duration: 45 min
Grade focus: Grade 1–6
Materials (Unplugged): Floor grid + any obstacles (chairs, cups, bags) · (Robot): Floor robot + real obstacles

Learning Objectives

  • Combine sequencing and obstacle-awareness into a full programme
  • Iterate: test, identify failure point, correct and re-test
  • Experience the full coding cycle: plan → code → run → debug → improve

📝 Unplugged Version

  • Set up an obstacle course using chairs and bags on the floor grid
  • One learner is the Robot — can only follow written instructions, cannot see ahead
  • Programmer must plan a safe route BEFORE giving any commands
  • If the Robot hits an obstacle, go back to step 1 — debug!

🤖 With Kit Version

  • Set up 2–3 real obstacles on the floor grid
  • Groups plan route on app first, then programme the robot
  • First group to get their robot to the goal without hitting anything wins — but the real prize is the fastest debugger!
  • Take photos/video for the school's social media 📸
End-of-Week Reflection: Learners complete the sentence "A robot is like a computer because..." and "A robot is different from a computer because..."
